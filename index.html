gateling使用总结

安装 官方文档

使用IntelliJ IDEA

  1. 安装Scala插件： 点击【IntelliJ IDEA】 -> 【Preferences】 -> 【Plugins】，搜索 “Scala”，搜索到插件然后点击底部的 【Install JetBrains plugin…】安装重启即可。

  2. 创建maven项目： file => new => project => Maven => Click【Create from archetype】查看最新的archetype =》 next =》next =》next =》 finish。

  3. 项目目录说明：

  • bodies：用来存放请求的body数据
  • data：存放需要输入的数据
  • scala：存放Simulation脚本
  • Engine：同运行bin\gatling.sh效果一致
  • Recorder：同运行跟运行bin\recorder.sh 效果一致，录制的脚本存放在scala目录下
  • target：存放运行后的报告

配置文件

  1. gatling.conf
  2. 日志logback.xml 官方文档Debugging
  • logger name=”io.gatling.http.engine.response” level=”TRACE”（输出response）

所在目录：target/gatling/类名-时间戳

  1. 报告：index.html
  2. 数据：simulation.log
  3. 60个用户请求在180s内平顺的注入。主要是看系统随着用户增多，响应时间是否成直线上升状况。我们比较关心每个用户请求链的时间，官方报告没有具体给出，所以对写程序对simulation.log数据文件进行分析。
